Day14 重点研究一下input的数据中包含的内容都有哪些 以及怎么把一个可用的数据变成他的格式来进行传入

昨天其实以及找了点时间看了一下在这个input文件中，即这个.mat文件中有哪些数据，具体的内容见下图所示：

可以看到其中有一些数据是可以对应联系起来的，比如rest_frames和move_frames是在同一个判定规则下得到的rest或者move状态的二分类，两个数据加起来是93191+446809=540000 和总的帧数相符，而rest_frames_fast和move_frames_fast 应该是另一个规则下的产物，而这两个值相加的总和$283639+256052< 540000$ 说明在这个分类过程中，有一些帧是被舍弃掉的，但是具体舍弃的规则是什么，目前在这还没看到

对于一些其他的数据可以做一些猜测 比如aligned_mean_position数据量是5400003 可以认为是对每帧找了个所有点的平均位置（可能是质心或者标记中心）来表示小鼠在整个环境中的位置。同样还有个aligned_rotation_matrix 是22*540000的矩阵，猜测是不是以四元组的格式还是一些别的存法（因为在matlab中没办法打开这样太大的文件，所以看不到里面具体是什么格式）还有一些bad_frames相关的数据，应该是对于一些判定下得到的不好的帧，会在一些操作中被忽略使用，同时被记录下来。

粗略的读了以上数据后，我觉得光这样看其实并不能完全理解到这些数据的具体含义和作用，因此我觉得还是得回到代码中，从一个正向的过程看一下这个输入的数据是怎么被读的，读的过程中做了哪些操作，数据中的哪些内容被用了。

正向代入代码

先看“CAPTURE_quickdemo.m”这个代码里的内容，首先从这段代码中读取了这个文件：

%load mocap file：
这是一条注释，指示代码的主要操作是加载运动捕捉（mocap）数据文件。

if isempty(inputfile)：
这行代码检查变量 inputfile 是否为空。如果为空，表示没有指定输入文件，进入下面的条件分支。

datafile = ...：接下来的代码将从指定路径加载运动捕捉数据文件。具体路径是硬编码的，不是通过 inputfile 来指定的。这个路径可能是 ‘F:\rats\CAPTURE_demo-master\Data\nolj_Recording_day8_caff1_nolj_imputed.mat’。
else：

如果 inputfile 不为空，表示有指定的输入文件，则进入这个分支。

datafile = load(inputfile);：从指定路径加载运动捕捉数据文件，并将其内容存储在变量 datafile 中。

if isstruct(datafile) && numel(fieldnames(datafile)) == 1：检查 datafile 是否是一个结构体且包含一个字段。

fname = fieldnames(datafile);：如果是结构体，获取它的字段名。

datafile = datafile.(fname{1});：将结构体中的数据提取出来，存储在 datafile 中。

mocapstruct = datafile;：
最后，将加载的数据存储在 mocapstruct 变量中，以供后续使用。这个变量可能包含了运动捕捉数据的各种信息，例如帧数据、标记点信息等。

mocapstruct.modular_cluster_properties.clipped_index{8} = 1:size(mocapstruct.aligned_mean_position,1 );：

这行代码设置了 mocapstruct 结构体中的 modular_cluster_properties.clipped_index{8}。感觉是在确定要使用的帧范围。具体地，size(mocapstruct.aligned_mean_position, 1) 返回了 mocapstruct.aligned_mean_position 矩阵的行数。1:size(mocapstruct.aligned_mean_position, 1) 生成了一个数字序列，从1开始，以1为步长，一直增加到矩阵行数的值,然后把这个值赋给modular_cluster_properties中的clipped_index{8}这个属性，可能是用来控制要选择的帧数，这可能是为了指定要用于行为特征计算的帧数据的范围。

if ~exist(MLmatobjfile,'file') || overwrite_MLmatobjfile：

这行代码检查一个名为 MLmatobjfile 的文件是否存在，以及是否设置了 overwrite_MLmatobjfile 标志。如果文件不存在或者设置了 overwrite_MLmatobjfile 为真（非零），则进入下面的条件分支。这里的情况就是这个文件不存在，所以要用下面的代码来调用create_behavioral_features来生成

MLmatobj = create_behavioral_features(mocapstruct, coefficient_file, overwrite_coefficient, linkname);：

这里调用了一个名为 create_behavioral_features 的函数，传递了多个参数：
mocapstruct：运动捕捉数据的结构体(即前面读入的inputfile)，用于计算行为特征。
coefficient_file：系数文件的名称，可能包含了用于计算行为特征的一些参数。
overwrite_coefficient：一个标志，指示是否覆盖现有的系数文件。
linkname：一个参数，可能是用于行为特征计算的链接名称或标识符。
这里的coefficient_file好像也是在运行过程中生成的，这边也需要研究一下是如何生成的，初步认为在create_behavioral_features这个函数调用时，这里的coefficient_file是传入一个文件名，同样也是在使用这个函数的时候创建的系数文件。

create_behavioral_features

进入这个文件看一下，这里的主要设置如下：

前面的MLmatobj在这个文件中的计算就是这个ML_features 这里可以看到主要是调用了三个函数来进行计算，分别是compute_joint_angles_demo,compute_appendage_pc_demos 和 compute_wl_transform_features_demo
从命名来看 第一个是计算了一些关节角的特征 第二个是计算了一些四肢的特征（pc具体是是什么还不知道） 最后一个函数是计算一点小波变换的特征

接下来进入这几个函数看看具体是怎么用的

compute_joint_angles_demo

ML_features = compute_joint_angles_demo(mocapstruct,linkname);
这一行代码传了两个参数 分别是mocapstruct 也就是前面的input数据 linkname是输入的“myrat” 表明是在计算小鼠的特征，这倒不是很重要，那进入compute_joint_angles_demo这个文件看看mocapstruct中的哪些内容被使用了

ML_features = struct();：

这行代码创建一个空的 MATLAB 结构体，命名为 ML_features，用于存储关节角度特征。

fprintf('computing joing angles \n');：

这是一行注释，用于指示代码将要计算关节角度特征。

注释部分：

在注释部分，代码提供了关于姿势角度的定义和术语。它介绍了一些用于描述动物姿势的坐标系，例如：

“saggital” 表示动物的冠状面（平行于动物的长轴）。
“coronal” 表示动物的横截面（与冠状面垂直）。
“transverse” 表示动物的横截面（与前两者垂直）。
“allangles” 包含所有的角度。
anglestruct = load_mouse_kyle_anglestruct();：

这行代码调用了一个函数 load_mouse_kyle_anglestruct()，并将结果存储在 anglestruct 变量中。这个函数的目的可能是加载一些与关节角度计算相关的参数或定义，以便后续的计算。这里读了一下对应的这个文件，其中基本上就是写了一些关节的名字以及对应的关节对组成，如图所示：

switch linkname：

这是一个 switch 语句，用于根据不同的 linkname 值加载不同的角度结构体。linkname 可能是一个表示不同动物类型（如 “rats”、”bird”、”mouse” 等）的字符串。

segment_pairs, coronal_pairs, saggital_pairs, transverse_pairs, planar_trios：

这些变量是从不同的角度结构体中加载的，包含了与角度计算相关的关节段、平面对、和角度计算所需的参数和定义。

这段代码的主要目的是计算关节角度并将结果存储在 jointangle_struct 结构体中。以下是代码的详细说明：

% load the variables in from the structfield：

这是一段注释，可能是用于提醒代码将要从 anglestruct 结构体中加载一些变量。

assigns = structvars(anglestruct);：

这行代码调用了 structvars 函数来获取 anglestruct 结构体中的变量名称，并将结果存储在 assigns 中。这个操作的目的可能是为了在后续的循环中逐个加载这些变量。

for kk = 1:size(assigns, 1)：

这是一个 for 循环，用于逐个加载 anglestruct 结构体中的变量。

eval(assigns(kk, :))：这行代码使用 eval 函数执行字符串表达式，其中 assigns(kk, :) 包含了要加载的变量名称。这样，代码会逐个加载 anglestruct 中的变量。
jointangle_struct = struct();：

这行代码创建一个空的 MATLAB 结构体 jointangle_struct，用于存储关节角度数据。

all_seglengths, all_segments, transverse_seglengths：

这些变量是用来存储关节角度计算中的一些中间结果或临时数据的单元格数组。

ML_features.segment_pairs 和 ML_features.include_angles：

这些字段用于存储关节角度计算中的一些参数和定义。

两个 for 循环：

第一个 for 循环用于计算横断面（coronal）方向的关节角度，通过计算向量之间的夹角来实现。
第二个 for 循环用于计算冠状面（saggital）方向的关节角度，同样也是通过计算向量之间的夹角来实现。
在这两个循环中，代码使用运动捕捉数据中的标记点位置（mocapstruct.markers_aligned_preproc）(20个markers的位置，都是540000*3)来计算关节角度。计算涉及了向量的差异、点积和三角函数等数学操作。计算的结果存储在 jointangle_struct 结构体中，每个关节角度对应于不同的命名字段，例如 saggital_names 和 coronal_names。

总的来说，这段代码实现了从运动捕捉数据中计算关节角度，并将这些角度数据存储在 jointangle_struct 结构体中，以供后续的数据分析和处理使用

这段代码完成了以下工作：

计算横断面（transverse）方向的关节角度：

使用 for 循环遍历横断面方向的不同关节对（transverse_pairs）。
计算每对关节的角度，通过计算两个向量之间的夹角，这些向量是由运动捕捉数据中的标记点位置计算而来的。
将计算得到的关节角度存储在 jointangle_struct 结构体中的相应字段中。

计算横断面方向的关节段长度：

使用 for 循环遍历横断面方向的不同关节对（transverse_pairs）。
计算每对关节段的长度，通过计算两个标记点之间的欧氏距离。
将计算得到的关节段长度存储在 transverse_seglengths 单元格数组中。

计算平面关节角度（planar joint angles）：

使用 for 循环遍历不同的平面关节角度组合（planar_trios）。
根据每个平面组合的定义，调用 get_planar_jointangles 函数来计算相应的关节角度。
根据平面关节角度组合的不同情况，将计算得到的角度存储在 jointangle_struct 结构体的相应字段中。

计算关节段长度：

使用 for 循环遍历不同的关节对（segment_pairs）。
计算每对关节段的长度，通过计算两个标记点之间的欧氏距离。
将计算得到的关节段长度存储在 all_seglengths 单元格数组中，同时还将向量差异存储在 all_segments 单元格数组中。

数据处理和清理：

去除关节角度数据中的虚部，确保角度数据为实数。
将角度数据中的 NaN 值替换为零。
计算关节角度的平均值：

使用 structfun 函数计算 jointangle_struct 结构体中各个字段的平均值，并将结果存储在 ML_features.joint_angles_mean 中。

在这一段计算中，主要使用到了markers_aligned_preproc和modular_cluster_properties

compute_appendage_pc_demos

这段代码执行了以下操作：

加载或初始化系数结构（coeffstruct）：

如果指定的系数文件 coeffstruct_in 存在且 overwrite_coeff 为0（不覆盖），则尝试从文件中加载系数结构。
如果加载失败，则尝试重新加载。如果文件不存在或需要覆盖，则初始化一个空的系数结构 coeffstruct。这里他传入的参数是“demo_coefficient” 这个命名的文件在这个时候应该还没有创建，因此调用的应该是else后面的初始化一个空的系数结构coeffstruct

指定不同附属肢体的角度：

定义了附属肢体的名称，存储在 appendage_names 中，包括 ‘Head’, ‘axial’, ‘LArm’, ‘Rarm’, ‘LLeg’, ‘RLeg’, ‘’, ‘globall’, ‘trunk’。
初始化了存储附属肢体角度的变量，包括 appendage_anglegps 和 appendage_segvals。

获取用于主要 t-SNE 映射的角度和帧索引：

指定了附属肢体的组索引为8，存储在 appendage_gps 中。
获取每个附属肢体组的帧索引，存储在 frames_appendage_gps 中。
这里把modular_cluster_properties.clipped_index{8}的数据依次写到frames_appendage_gps中，把所有的都修改为8% mocapstruct.move_frames。

高通滤波和剪裁所有关节角度：

定义了高通滤波的参数，包括 params.fps、params.difforder、params.medfiltorder 和 params.gaussorder。
获取每个附属肢体组的关节角度数据，并进行高通滤波和剪裁。

执行主成分分析（PCA）：

对每个附属肢体组的关节角度进行PCA分析，得到主成分系数、解释方差等。
如果指定的系数结构中不存在主成分系数或需要覆盖，则进行PCA分析并将结果存储在 COEFFS_appendages 中。
执行主成分分析（PCA）：针对关节长度数据执行与步骤5类似的PCA分析。

计算欧氏距离的主成分分析（PCA）：对每个附属肢体组的欧氏距离数据执行PCA分析。

将PCA结果存储在 ML_features 结构中：

存储主成分系数、PCA得分和解释方差等信息，分别对应关节角度数据、关节长度数据和欧氏距离数据。
为每个附属肢体组存储相应的PCA结果。

后续的代码内容：

保存附属肢体的系数：

如果 save_coeffs 为真（1），则将附属肢体的系数结构 coeffstruct 保存到指定的文件 coeffstruct_in 中。

对平滑动态应用PCA：

针对附属肢体组中的每个角度，获取关节角度数据。
减去平均值并应用之前计算的主成分系数，得到关节角度的主成分。
剪裁NaN和无穷大值。
存储结果在 ML_features.appendage_joint_angles_pcs_hipass 中。

对欧氏距离数据应用PCA：

针对附属肢体组中的每个欧氏距离向量，获取欧氏距离数据。
减去平均值并应用之前计算的主成分系数，得到欧氏距离的主成分。
存储结果在 ML_features.appendage_pca_score_euc_hipassclip 中。

平滑化dyadic谱图：

如果 do_timescales 为真，则尝试不同的时间尺度（timescales）来平滑dyadic谱图数据。
使用高斯滤波器平滑附属肢体组的PCA得分。
存储平滑后的结果在 ML_features.appendage_pca_score 和 ML_features.appendage_pca_score_lengths 中，带有不同时间尺度的后缀。这部分代码目前被注释掉，需要根据需要取消注释。
